# AI 应用开发工程师 · 学习计划（修正版 v3）

## 定位修正

经实际项目分析，各方向真实水平：

| 维度 | 修正定位 |
|------|----------|
| Python | **初中级偏上**（能看懂高级语法但独立写需练习） |
| FastAPI | 理解路由/中间件/异步概念，缺少独立构建肌肉记忆 |
| RAG | **入门级，有生疏** — 代码能跑但需重新捡起 |
| LangGraph | **早期中级** ✅ React Agent、Agentic RAG、Reflection、Reflexion 四项目实战 |
| 综合 AI | 理论和动手均有基础，缺的是进阶工程能力（HITL、流式、记忆、多Agent） |

核心策略：
1. **第 1~2 周** — 脱离 AI 手写代码，把理解变成肌肉记忆
2. **第 3~4 周** — RAG 重温 + 升级（重排序、查询改写、评估）
3. **第 5 周** — LangGraph 进阶 + Deep Agents 双轨学习
4. **第 6 周** — 终极项目

---

## 总周期：6~8 周

---

### 第 0 周 — 摸底过渡（2~3 天）

跑通两个老项目，唤醒记忆：

- **local-llm-fast-api** — 逐文件看一遍，理解每层职责
- **document-helper** — 重新跑通 ingestion → 检索 → 问答全链路

---

### 第 1~2 周 — Python 手感复健（无 AI 辅助）

**练习 1：手写最小 FastAPI 服务**
- GET /hello、POST /echo（Pydantic 校验）、请求耗时中间件

**练习 2：手写异步调用**
- httpx.AsyncClient + asyncio.gather + Semaphore 限流

**练习 3：手写 LLM 流式调用**
- httpx 流式读取 LM Studio API，async for chunk yield

**验收标准：** 不用 AI，20 分钟内写出带错误处理的流式 LLM 调用函数。

---

### 第 3 周 — RAG 重温

- 跑通 document-helper，理解每一行代码
- 修掉拼写错误、删掉死代码
- 手写一遍 ingestion 核心逻辑（分割 + 向量化 + 入库）
- 手写一遍检索函数

---

### 第 4 周 — RAG 升级

- 增加**重排序**（sentence-transformers cross-encoder）
- 增加**查询改写**（LLM 把问题改写成 3 个检索语句）
- 写 **RAG 评估脚本**（命中率、回答相关性）

**验收标准：** 能跟面试官聊清楚"为什么这样设计，还有什么可以优化"。

---

### 第 5 周 — LangGraph 进阶 + Deep Agents 双轨学习 🔥

Deep Agents 是 LangChain 官方推出的 Agent Harness（代理套件），底层基于 LangGraph 运行时，开箱即提供 HITL、记忆、子Agent、流式等能力。

双轨学习法：上午手写 LangGraph 理解底层原理（面试能聊），下午体验 Deep Agents 感受开箱效率（工作能干活）。

| 天 | 上午 — LangGraph 手写 | 下午 — Deep Agents |
|---|----------------------|-------------------|
| 1~2 | 手写 Human-in-the-loop（interrupt + resume） | 看 Deep Agents HITL 文档，对比实现 |
| 3 | 手写 MemorySaver + SQLite Checkpointer | 看 Deep Agents Memory 文档，对比实现 |
| 4 | 手写 Supervisor + Worker 多 Agent | 看 Deep Agents Subagents 文档，对比实现 |
| 5 | 手写流式 Streaming + SSE | 看 Deep Agents Streaming 文档，对比实现 |
| 周末 | **对比项目**：同一个任务分别用 LangGraph 手写和 Deep Agents 实现 |

---

### 第 6 周 — 终极项目：多 Agent 代码助手

```
Supervisor Agent
  ├── 代码阅读 Agent（读文件 + RAG 检索）
  ├── 代码执行 Agent（安全沙箱跑命令）
  └── 代码审查 Agent（结构化分析 + 评分）

Human-in-the-loop: 关键操作需人工确认
流式输出: 实时推送推理过程
持久化记忆: 记住项目结构和历史对话
```

## 时间线总览

```
第 0 周  ├── 跑通两个老项目，唤醒记忆
第 1~2 周 ├── 手写 Python 复健（无 AI 辅助）
第 3 周  ├── RAG 重温 + 手写核心逻辑
第 4 周  ├── RAG 升级（重排序 + 查询改写 + 评估）
第 5 周  ├── LangGraph 进阶 + Deep Agents 双轨
第 6 周  └── 终极项目：多 Agent 代码助手
```

## 面试准备建议（最后 3 天）

1. **RAG 详解** — 分块策略、检索优化、重排序的必要性
2. **Agent 设计模式** — ReAct 循环、工具调用、状态管理
3. **LangGraph vs Deep Agents** — 知道运行时和 Harness 的区别
4. **Human-in-the-loop** — 为什么需要、如何实现
5. **多 Agent 编排** — Supervisor/Worker、图工作流
6. **前端+AI** — 你的差异化优势（Vue + 流式 SSE）
