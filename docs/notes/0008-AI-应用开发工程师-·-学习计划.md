# AI 应用开发工程师 · 学习计划

## 定位修正

经项目分析发现，实际水平高于自我评估：

| 维度 | 修正定位 |
|------|----------|
| Python | **初中级偏上**（看懂高级语法 > 独立写出，需手写复健） |
| FastAPI | 理解路由/中间件/异步等概念，但缺少独立构建肌肉记忆 |
| RAG | **入门级，有生疏** — 代码能跑但需要重新捡起 |
| AI 整体 | **理论和读码能力 > 动手编码能力** |

核心策略：**不要 AI 辅助，手写代码把理解变成肌肉记忆**

---

## 总周期：6~8 周

---

### 第 0 周 — 摸底过渡

花 2~3 天分别跑通两个项目，唤醒记忆：

- **local-llm-fast-api** — 启动项目，逐文件看一遍，在注释里写下每层职责
- **document-helper** — 重新跑通 ingestion → 检索 → 问答全链路

**目的：** 不是新学，是唤醒记忆。

---

### 第 1~2 周 — Python 手感复健

**关键阶段** — 脱离 AI，自己手写。

**练习 1：手写最小 FastAPI 服务**
- `GET /hello`
- `POST /echo` 接收 JSON 返回 JSON
- Pydantic 模型校验
- 中间件记录请求耗时

**练习 2：手写异步调用**
- `httpx.AsyncClient` 并发请求两个 URL
- `asyncio.gather` + `Semaphore` 限流

**练习 3：手写 LLM 流式调用**
- `httpx` 流式读取 LM Studio API
- `async for chunk in response.aiter_bytes()`
- `yield chunk` 返回

**验收标准：** 不用 AI，20 分钟内能写出一个带错误处理的流式 LLM 调用函数。

---

### 第 3~4 周 — RAG 重温 + 升级

**第 3 周 — 重温 RAG**
- 跑通 document-helper，理解每一行代码
- 修掉拼写错误、删掉死代码
- 手写一遍 `ingestion.py` 核心逻辑（分割 + 向量化 + 入库）
- 手写一遍检索函数

**第 4 周 — 升级 RAG**
- 增加**重排序**（sentence-transformers cross-encoder）
- 增加**查询改写**（LLM 把问题改写成 3 个检索语句）
- 写 **RAG 评估脚本**（计算命中率、回答相关性）

**验收标准：** 能跟面试官聊清楚"为什么这样设计，还有什么可以优化"。

---

### 第 5~6 周 — Agent 深度实战

**第 5 周 — 从零手写最小 Agent（不用框架）**
```python
# 1. LLM 调用 → 2. 解析 function_call → 3. 执行工具 → 4. 返回结果 → 循环
# 就两个工具：get_weather() 和 search_web()
```
从零写过，对 Agent 原理一清二楚。

**第 6 周 — LangGraph 多 Agent + 终极项目**
- StateGraph 有状态工作流
- Supervisor + Worker 模式
- Human-in-the-loop 中断+恢复
- 对接第 4 周升级好的 RAG 系统

**终极项目：** "AI 代码助手"Agent — 能读项目文件、执行命令、搜 RAG 知识库。

---

## 时间线总览

```
第 0 周  ├── 跑通两个项目，唤醒记忆
第 1 周  ├── 手写 FastAPI 基础（无 AI）
第 2 周  ├── 手写异步 + 流式 LLM（无 AI）
第 3 周  ├── 重温 RAG，手写核心逻辑
第 4 周  ├── 升级 RAG（重排序 + 查询改写 + 评估）
第 5 周  ├── 从零手写最小 Agent（不用框架）
第 6 周  └── LangGraph 多 Agent + 对接 RAG = 终极项目
```

## 面试准备建议（最后 3 天）

重点准备方向：
1. **RAG 详解** — 分块策略、检索优化、重排序的必要性
2. **Agent 设计模式** — ReAct 循环、工具调用、状态管理
3. **Token 计算** — 理解上下文窗口和成本
4. **幻觉处理** — 减少、检测、兜底策略
5. **系统提示设计** — 角色设定、few-shot、约束条件
6. **加分项** — 前端 + AI 整合能力是你的差异化优势
