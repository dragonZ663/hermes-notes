# Shell 重定向详解：>、2>&1 与日志记录

## 一、命令全貌

```bash
python script.py > log.txt 2>&1
```

这条命令的作用：**执行 Python 脚本，并把所有输出（正常输出 + 错误输出）统一写入一个日志文件。**

---

## 二、各部分拆解

### 1. `python script.py`

用 `python` 执行指定的脚本文件。

### 2. `> log.txt` — 标准输出重定向

- `>` 把 **标准输出（stdout）** 写入文件
- Python 里的 `print()` 输出默认到终端，加上 `>` 后写入日志文件
- **注意：** `>` 会覆盖原文件内容。如果想追加而不是覆盖，用 `>>`

```bash
# 覆盖写入
python script.py > log.txt

# 追加写入
python script.py >> log.txt
```

### 3. `2>&1` — 标准错误合并

这是很多人容易疑惑的部分。

- `2` = **标准错误输出（stderr）**
- `1` = **标准输出（stdout）**
- `2>&1` = **把 stderr 重定向到 stdout 当前指向的位置**

因为前面已经有 `> log.txt`，stdout 已经指向日志文件，所以 `2>&1` 的效果是：**错误信息也一起写入同一个日志文件**。

#### 典型场景

脚本内容：
```python
print("start")
raise Exception("something wrong")
```

运行 `python test.py > test.log 2>&1` 后，终端无输出，`test.log` 中包含：
```
start
Traceback (most recent call last):
  ...
Exception: something wrong
```

---

## 三、顺序为什么重要

### ✅ 正确写法

```bash
python script.py > log.txt 2>&1
```

执行顺序：
1. 先把 stdout 重定向到 `log.txt`
2. 再让 stderr 跟随 stdout
3. 结果：**stdout 和 stderr 都进 log.txt** ✅

### ❌ 错误写法

```bash
python script.py 2>&1 > log.txt
```

执行顺序：
1. 先把 stderr 重定向到当前 stdout（通常是终端）
2. 再把 stdout 重定向到 `log.txt`
3. 结果：**stdout 进文件，stderr 还在终端** ❌

## 四、常见变体

### 丢弃所有输出

```bash
python script.py > /dev/null 2>&1
```

`/dev/null` 是 Linux 的**黑洞**——进去的数据直接消失。常用于不关心输出的后台任务。

### 后台持久运行

```bash
nohup python script.py > log.txt 2>&1 &
```

- `nohup` — 即使退出终端，进程也不终止
- `&` — 放到后台执行
- 组合效果：**脚本在后台持续运行，所有输出写入日志，不受终端断开影响**

### 同时输出到终端和文件

```bash
python script.py 2>&1 | tee log.txt
```

`tee` 命令把输出**同时送到终端和文件**。

## 五、日志查看命令

| 命令 | 用途 |
|------|------|
| `cat log.txt` | 查看全部内容 |
| `less log.txt` | 分页查看 |
| `tail -f log.txt` | 实时追踪最新输出 |
| `grep "error" log.txt` | 搜索关键词 |

---

## 六、一句话总结

> `python script.py > log.txt 2>&1` = **运行脚本，把 stdout 和 stderr 都写进同一个日志文件。顺序是先重定向 stdout，再让 stderr 跟随，缺一不可。**
